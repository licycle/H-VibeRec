# H-VibeRec macOS 安装与首次配置

这份文档面向非开发者。安装后主要跟着 H-VibeRec 设置页里的按钮操作即可。

## 快速摘要

- 安装方式：从 GitHub Releases 下载 `.dmg`，打开后把 `H-VibeRec.app` 拖到 `Applications`。
- 首次配置入口：打开应用右上角设置，按 `检测`、`基础配置`、`语音输入法` 三个页面的按钮完成配置。
- 必做：在 `检测` 页面点击 `验证录音权限`、`检测运行环境`、`下载/检查 FunASR workflow 模型`。
- 可选：在 `基础配置` 页面填写兼容 OpenAI Chat Completions API 的 LLM 配置；在 `语音输入法` 页面启用全局语音输入。

## 安装应用

1. 打开项目的 GitHub Releases 页面。
2. 下载最新的 `.dmg` 文件。
3. 双击打开 `.dmg`。
4. 把 `H-VibeRec.app` 拖到 `Applications` 文件夹。
5. 从 `Applications` 启动 H-VibeRec。

如果 macOS 首次启动时拦截应用，在 `Applications` 中右键点击 `H-VibeRec.app`，选择 `Open`。如果仍被拦截，按 macOS 弹窗或安全设置里的提示允许打开。

## 第一次打开后怎么配置

打开 H-VibeRec 后，点击设置按钮。按下面顺序操作即可。

### 1. 在“检测”页完成基础检查

进入 `检测` 页面，依次点击：

1. `验证录音权限`
2. `检测运行环境`
3. `下载/检查 FunASR workflow 模型`

`验证录音权限` 会检查麦克风和系统音频权限。如果权限没有通过，页面会出现对应按钮，例如 `打开麦克风权限设置` 或 `打开系统音频权限设置`。点击这些按钮，按系统弹窗允许 H-VibeRec，然后回到应用再次点击 `验证录音权限`。

`下载/检查 FunASR workflow 模型` 会准备本地转写模型。等待进度完成，直到模型状态显示已就绪。

### 2. 在“基础配置”页配置总结和问答

本地录音和本地转写不需要 API Key。下面这些功能需要 LLM：

- 会议纪要
- 长文本总结
- 本地笔记 AI 问答
- 语音输入法的 AI 润色模式

如果需要这些功能，进入 `基础配置` 页面：

1. 填写你的 OpenAI-compatible Chat Completions API 服务配置：`LLM Provider`、`LLM Base URL` 和 `LLM Model`。
2. 在 `API Key` 中粘贴你的 key。
3. 点击 `保存配置`。
4. 点击 `测试 LLM`。

API Key 会保存在系统钥匙串（keychain）。`API Key` 输入框留空保存时，不会修改已经保存的 key。

如果模型下载失败，再回到 `基础配置` 页面切换 `模型下载源`，或按你的网络情况填写代理字段，然后保存配置并重新点击 `下载/检查 FunASR workflow 模型`。

### 3. 可选：启用“语音输入法”

只有需要用全局快捷键把语音写入当前输入框时，才需要配置这个页面。

进入 `语音输入法` 页面：

1. 打开 `启用语音输入法`。
2. 设置或保留默认全局快捷键。
3. 点击 `保存配置`。
4. 点击 `检查权限`。

如果权限没有通过，页面会显示 `请求辅助功能授权`、`打开辅助功能权限设置` 或 `打开麦克风权限设置`。按页面显示的按钮处理即可。

## 开始使用

1. 创建或选择一个本地空间。
2. 点击录音按钮开始录音，或导入已有音频。
3. 对音频执行本地转写。
4. 查看带时间戳和说话人标记的转写文本。
5. 选择总结模板生成会议纪要。
6. 在右侧 AI 问答面板中基于本地笔记提问。

## 本地数据位置

默认数据目录：

`~/Documents/Voice Vibe Local`

常见内容：

- `app.sqlite`：本地数据库。
- `models/`：本地 ASR 模型。
- `workspaces/`：本地工作空间。
- `transcripts/`：转写输出。
- `summaries/`：总结输出。

## 常见问题

### 没有录到声音

打开设置，进入 `检测`，点击 `验证录音权限`。如果页面出现权限按钮，点击按钮并按系统提示允许 H-VibeRec。

### 系统声音没有录进去

打开设置，进入 `检测`，点击 `验证录音权限`。如果页面出现 `打开系统音频权限设置`，点击它并完成授权，然后重启应用。

### 模型下载慢或失败

进入 `基础配置`，切换 `模型下载源`，或填写代理字段并点击 `保存配置`。然后回到 `检测`，重新点击 `下载/检查 FunASR workflow 模型`。

### LLM 测试失败

进入 `基础配置`，检查 `LLM Base URL`、`LLM Model` 和 `API Key`，保存后再点击 `测试 LLM`。

### 语音输入法无法写入当前输入框

进入 `语音输入法`，点击 `检查权限`，按页面出现的按钮完成授权。密码框、安全输入、阻止粘贴的应用、远程桌面、虚拟机和部分游戏窗口可能无法写入。
