# ASR Runtime

`runtime/asr` is generated by `npm run runtime:ensure` and bundled into the Tauri app.

It must contain everything required to run the FunASR workflow sidecar except model weights:

- `bin/python` on macOS, or `python.exe` on Windows
- Python packages from `sidecars/funasr_nano_mlx/requirements.txt`
- `bin/ffmpeg` on macOS, or `bin/ffmpeg.exe` on Windows

The bundle must also include:

- `sidecars/funasr_nano_mlx/main.py`
- `sidecars/funasr_nano_mlx/requirements.txt`

Do not commit generated runtime files. The app intentionally does not fall back to system Python or system ffmpeg.

Model weights are the only runtime assets downloaded after installation. They are prepared from the settings page button. The ASR repo is downloaded to the app models directory, and VAD/CAM++/punctuation models are stored beside it under `.voice_vibe_aux` or under the shared auxiliary directory when a custom ASR model path is configured.

The workflow model set is:

- `paraformer-zh` (`v2.0.4`)
- `fsmn-vad` (`v2.0.4`)
- `cam++`
- `ct-punc-c` (`v2.0.4`)

Before publishing a package, run:

```bash
npm run runtime:check
npm run package:check
npm run bundle:verify-runtime
```

`package:check` is the fast packaging readiness check. It does not create an installer; it verifies the packaging contract before build. `bundle:verify-runtime` checks the built `.app` resources after `npm run tauri build`.

Both checks must pass before the app is considered installable, because installed builds are expected to run local ASR after model weights are downloaded.

Current platform status:

- macOS Apple Silicon: runtime generation and bundle verification are supported and have been exercised locally.
- Windows x64: runtime generation and packaging checks are implemented, but must be exercised on a Windows x64 machine before release.
- Intel macOS and Linux: no installable offline ASR package target yet.
